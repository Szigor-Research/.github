# Contributing

Contributions to public LASZLO Quantification repositories should preserve the
published trust boundaries.

- Use synthetic data and identifiers only.
- Do not include credentials, private endpoints, provider inventories, wallet data, or private-source copies.
- Add tests for contract, policy, state, or decision changes.
- Update security and architecture documents when a trust boundary changes.
- Keep product claims aligned with implemented behavior.

Use the repository's own contribution guide for development commands and
project-specific review requirements.
